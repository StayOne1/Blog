Git学习

Git 是一个分布式版本控制系统，可以便捷高效地处理任何项目。Git的内容存储使用的是SHA-1哈希算法。

Git 工作流程

  • 克隆 Git 资源作为工作目录。
  • 在克隆的资源上添加或修改文件。
  • 如果其他人修改了，你可以更新资源。
  • 在提交前查看修改。
  • 提交修改。
  • 在修改完成后，如果发现错误，可以撤回提交并再次修改并提交。

Git工作区、暂存区、版本库

  • 工作区： 就是你在电脑里能看到的目录
  • 暂存区： (index) 一般存放在.git目录下的index文件中，也叫做索引
  • 版本库： 工作区一个隐藏目录.git

创建版本库

  • 创建文件夹

    mkdir 文件夹名

  • 切换到某目录下

    cd 文件夹名

  • 查看当前目录

    pwd

  • 把当前目录初始化到仓库

    git init

  • 添加文件到仓库 可以添加多个文件

    git add 文件名

  • 提交到仓库

    git commit -m "解释"

时光机

  • 当前仓库状态(本地)
    git status
  • 查看文件上次修改记录
    git diff 文件名

版本回退

  • 显示最近到最远的提交日志
    git log
  • 简易显示
    git log --pretty=oneline

HEAD表示当前版本，HEAD^表示上一版本

HEAD^^表示上上版本，HEAD~100表示上100个版本

  • HEAD也可以换成是对应版本的前部分版本号

    git reset --hard HEAD^

  • 记录了每一次命令

    git reflog

连接远程仓库

  1. 生成ssh密钥

    ssh-keygen -t rsa -C "youremail@example.com"

    生成的密钥默认位于C:\Users\用户名.ssh文件夹中 默认文件名为id_rsa

  2. 建立与github的连接

    进入用户settings ，选择ssh and GPG keys,创建一个ssh密钥，把公钥(.pub)粘贴进去

  3. 连接远程仓库

    在github中创建项目仓库，并复制仓库ssh链接 通常为git@github.com:xxx/xxx.git

    在本地仓库进入git bash ，运行命令git remote add origin git@github.com:xxx/xxx.git

  4. pull操作

    git pull --rebase origin 分支名

  5. push操作

    git push -u origin 分支名
